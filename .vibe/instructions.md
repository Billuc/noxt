# Vibe Instructions for Noxt Project

When contributing to this project, follow these instructions:

## Project Scope & Guidelines

Read `README.md` to understand the project purpose, structure, and technical stack.

Read `CONTRIBUTING.md` for:

- How to contribute (fork, branch, commit, PR process)
- Style guide requirements
- Documentation standards

## My Responsibilities

- **Follow the style guide**: Keep functions small, documented, and only comment unintuitive behavior
- **Update code directly**: Make changes directly to the codebase as requested
- **Update docs with behavior changes**: Modify README, JSDoc, or other docs as needed
- **Keep changes minimal**: Only modify what is requested; match existing patterns
- **Verify changes**: Run tests, verify builds, and confirm edits land correctly

## Before Making Changes

1. Read `CONTRIBUTING.md` for current project conventions
2. Read `README.md` for project context
3. Check existing code for usage patterns
4. Confirm requirements with the user if unclear

## During Changes

- Apply the style guide from `CONTRIBUTING.md`
- Document all functions with JSDoc
- Limit comments to non-obvious behavior
- Keep changes focused and atomic

## After Changes

- Verify edits by reading back modified files
- Confirm tests pass if applicable
- Remind the user to update documentation if behavior changed
